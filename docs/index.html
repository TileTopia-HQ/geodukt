Geodukt

A declarative geospatial ETL pipeline — dbt for spatial data. Define transforms as a DAG. Geodukt resolves dependencies and materializes outputs.

Rust DAG Engine Declarative TOML GeoJSON / CSV Extensible Transforms AGPL-3.0-or-later

Features

Declarative Pipelines

Define sources, transforms, and sinks in TOML. No imperative scripting — just declare what you want.

DAG Execution

Automatic dependency resolution via topological sort. Detect cycles, validate references, execute in order.

Spatial Transforms

Buffer, centroid, filter, clip, reproject, simplify, dissolve. Pluggable registry — add your own operations.

Multi-Format I/O

Read GeoJSON, CSV with coordinates. Write GeoJSON. Designed for GeoPackage, GeoParquet, FlatGeobuf extension.

CLI Tool

geodukt run, validate, graph, init. Fast feedback on your pipeline before execution.

Extensible Architecture

Implement TransformOp, SourceReader, or SinkWriter traits. Register new operations in the transform registry.

Validation First

Check your DAG for cycles, unknown references, and duplicate names before running. Fail fast, fix early.

Execution Reports

After each run, see feature counts per stage. Know exactly what flowed through your pipeline.

TileTopia Ecosystem

Pairs with Geokode for geocoding enrichment, Jung for symbology, and upcoming stream processing + AI extraction.

Pipeline Example

# geodukt.toml
[project]
name = "city-analysis"
version = "0.1.0"

[[source]]
name = "buildings"
format = "geojson"
path = "data/buildings.geojson"

[[transform]]
name = "commercial_only"
input = "buildings"
operation = "filter"
field = "type"
equals = "commercial"

[[transform]]
name = "centroids"
input = "commercial_only"
operation = "centroid"

[[sink]]
name = "output"
input = "centroids"
format = "geojson"
path = "output/commercial_centroids.geojson"

Quick Start

# Initialize a project
geodukt init my-pipeline
cd my-pipeline

# Edit geodukt.toml to define your pipeline
# Add data to data/ directory

# Validate the DAG
geodukt validate

# Run the pipeline
geodukt run

# Visualize execution order
geodukt graph
